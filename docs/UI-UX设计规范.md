## 1. 设计理念

### 1.1 核心体验定位

```
深夜咖啡馆 · 温暖探索 · 安静陪伴
```

如同深夜咖啡馆的灯光——温暖、不刺眼、让人放松

### 1.2 Apple设计语言融合

| Apple设计原则 | 本产品应用 |
|----------------|------------|
| Clarity（清晰） | 信息层次分明，一眼看到重点 |
| Deference（克制） | 界面服务于内容，不喧宾夺主 |
| Depth（深度） | 层次感和动效增加愉悦感 |

### 1.3 视觉关键词

- **温暖** - 深色主题配暖色调点缀
- **呼吸感** - 充足的留白，像在咖啡馆随手翻书
- **流畅** - 自然过渡的动效，如同翻阅一本好书
- **探索** - 鼓励发现，而非导向

## 2. 设计原则

### 2.1 六大原则

```
┌─────────────────────────────────────────────────────────┐
│                                                         │
│   1. 温暖优先    深色背景配暖光，不刺眼，适合夜间阅读  │
│                                                         │
│   2. 呼吸留白    充足的空间，让内容"呼吸"，不拥挤     │
│                                                         │
│   3. 层次分明    信息分层清晰，重要内容一目了然        │
│                                                         │
│   4. 流畅动效    自然过渡，如同行云流水                │
│                                                         │
│   5. 轻触可达    移动端单手操作，核心功能一键触达     │
│                                                         │
│   6. 隐喻自然    交互符合直觉，无需学习成本           │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

### 2.2 不做清单

```
❌ 不做：闪烁、高饱和度的警告色
✅ 做法：柔和的提示色，安静不打扰

❌ 不做：密集的信息列表
✅ 做法：留白+卡片，信息分组清晰

❌ 不做：强制引导/弹窗打断
✅ 做法：静默提示，用户主导操作

❌ 不做：复杂层级导航
✅ 做法：最多2层，核心内容一步到达
```

## 3. 色彩系统

### 3.1 主色调

**深夜咖啡馆主题** - 深邃的夜空蓝黑 + 温暖的琥珀光

```css
:root {
  /* 主背景 - 如同咖啡馆的幽暗灯光 */
  --bg-primary: #0D1117;        /* 深邃夜空 */
  --bg-secondary: #161B22;      /* 次级背景 */
  --bg-card: #21262D;            /* 卡片背景 */
  --bg-elevated: #30363D;        /* 悬浮层背景 */

  /* 核心色 - 温暖的琥珀光 */
  --accent-primary: #F5A623;    /* 琥珀主色 - 如同咖啡馆灯光 */
  --accent-secondary: #7C5CFF;  /* 紫罗兰 - 知识图谱/探索 */
  --accent-cool: #58A6FF;       /* 冰蓝 - RFID科技感 */

  /* 文字 - 舒适的阅读体验 */
  --text-primary: #E6EDF3;       /* 主要文字 - 柔和白 */
  --text-secondary: #8B949E;     /* 次要文字 */
  --text-tertiary: #6E7681;     /* 辅助文字 */

  /* 功能色 */
  --success: #3FB950;            /* 成功 - 柔和绿 */
  --warning: #D29922;            /* 警告 - 柔和黄 */
  --error: #F85149;              /* 错误 - 柔和红 */
  --info: #58A6FF;              /* 信息 - 冰蓝 */
}
```

### 3.2 色彩比例

```
┌────────────────────────────────────────────────────┐
│                                                    │
│   背景色 (80%)  ████████████████████  #0D1117     │
│                                                    │
│   卡片色 (15%)  ████████  #21262D                 │
│                                                    │
│   强调色 (5%)   ███  #F5A623                      │
│                                                    │
└────────────────────────────────────────────────────┘

遵循 Apple HIG 的 60-30-10 法则
```

## 10. 动效设计

### 10.1 动效原则

```
Apple Human Interface Guidelines 动效四原则：

1. 响应性 (Responsive)
   - 即时反馈，0-100ms内响应

2. 清晰性 (Clear)
   - 动效服务于功能，不喧宾夺主

3. 连贯性 (Coherent)
   - 动效符合物理直觉

4. 愉悦性 (Delightful)
   - 微妙的缓动增加愉悦感
```

### 10.2 缓动曲线

```css
/* Apple标准缓动曲线 */
--ease-default: cubic-bezier(0.25, 0.1, 0.25, 1);
--ease-in: cubic-bezier(0.42, 0, 1, 1);
--ease-out: cubic-bezier(0, 0, 0.58, 1);
--ease-in-out: cubic-bezier(0.42, 0, 0.58, 1);

/* 弹性动效 */
--ease-spring: cubic-bezier(0.175, 0.885, 0.32, 1.275);
```

### 10.3 时长规范

| 类型 | 时长 | 示例 |
|------|------|------|
| 微交互 | 100-200ms | 按钮点击、开关切换 |
| 页面过渡 | 300-400ms | 页面跳转、模态弹出 |
| 加载动画 | 持续 | 骨架屏旋转、脉冲 |
| 打字效果 | 50ms/字 | AI回复 |

### 10.4 关键动效设计

#### 卡片悬浮动效

```css
.card {
  transition: transform 0.25s var(--ease-out),
              box-shadow 0.25s var(--ease-out);
}

.card:hover {
  transform: translateY(-4px);
  box-shadow: 0 12px 40px rgba(0, 0, 0, 0.25);
}
```

#### 页面过渡动效

```
入场：translateX(100%) → translateX(0)
       opacity: 0 → 1
       duration: 350ms
       easing: var(--ease-out)

退场：translateX(0) → translateX(-30%)
       opacity: 1 → 0
       duration: 250ms
       easing: var(--ease-in)
```

#### 呼吸动效（AI聊伴悬浮球）

```css
.ai-float-btn {
  animation: breathe 3s var(--ease-in-out) infinite;
}

@keyframes breathe {
  0%, 100% {
    transform: scale(1);
    box-shadow: 0 4px 20px rgba(245, 166, 35, 0.3);
  }
  50% {
    transform: scale(1.05);
    box-shadow: 0 6px 30px rgba(245, 166, 35, 0.5);
  }
}
```
