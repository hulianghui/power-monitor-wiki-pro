## 🎨 页面设计规范

### 1. 首页 (index.html)

**布局结构**:
```
Header (固定顶部，72px 高)
├── Logo (⚡图标)
├── 导航菜单 (首页、RFID 测温、在线监测、知识图谱)
└── 搜索框 + 用户头像

Main
├── Hero 区域 (100px 上内边距)
│   ├── 品牌 Badge
│   ├── 主标题 (56px，渐变文字)
│   └── 副标题 (18px，次要文字)
│
├── 双板块区域 (60px 上内边距)
│   ├── 板块标题区
│   └── 2 列网格
│       ├── RFID 卡片
│       └── 监测卡片
│
└── 功能网格区 (60px 上内边距)
    ├── 功能网格标题
    └── 4 列网格
        ├── 知识图谱
        ├── 参数计算
        ├── 故障模拟
        └── 设备选型

Footer (60px 上内边距)
├── 公司 Logo + 名称
└── 版权信息 + Power By HLH
```

### 2. 技术领域页 (rfid.html / monitor.html)

**布局结构**:
```
Header (与首页一致)

Main
├── 页面头部
│   ├── 返回按钮 ←
│   ├── 页面标题
│   └── 页面简介
│
├── 导航标签栏 (Tab Navigation)
│   ├── 标签 1 (激活状态)
│   ├── 标签 2
│   ├── 标签 3
│   ├── 标签 4
│   ├── 标签 5
│   └── 标签 6
│
└── 内容区域
    └── 根据选择的标签显示对应内容
        ├── 子模块 1
        ├── 子模块 2
        ├── 子模块 3
        └── 子模块 4

Footer (与首页一致)
```

### 3. 功能页面 (graph.html / chat.html / calculator.html 等)

**布局结构**:
```
Header (与首页一致)

Main
├── 页面头部
│   ├── 返回按钮 ←
│   └── 页面标题
│
└── 功能区域
    └── 根据具体功能设计

Footer (与首页一致)
```

## 🎯 交互设计

### 页面切换逻辑

```javascript
// 首页 → 技术领域页
点击 RFID 卡片 → 跳转到 rfid.html
点击 监测卡片 → 跳转到 monitor.html

// 技术领域页 → 子模块详情页
点击导航标签 → 切换内容区域 (不跳转页面)
点击子模块 → 展开详细内容 (手风琴效果)

// 首页 → 功能页面
点击 知识图谱 → 跳转到 graph.html
点击 参数计算 → 跳转到 calculator.html
点击 故障模拟 → 跳转到 simulator.html
点击 设备选型 → 跳转到 selector.html

// AI 助手
点击悬浮按钮 → 跳转到 chat.html
```

### 导航激活状态

```javascript
// 当前页面对应的导航项高亮
首页 → "首页" 高亮
RFID 页 → "RFID 测温" 高亮
监测页 → "在线监测" 高亮
知识图谱 → "知识图谱" 高亮
```

## 📋 内容组织原则

### 1. 信息密度

```
首页：低密度 (吸引注意力，提供入口)
技术领域页：中密度 (概览性信息，引导深入)
子模块页：高密度 (详细技术资料)
```

### 2. 导航深度

```
最多 3 次点击到达任何内容:
首页 → 技术领域页 → 子模块 → 详细内容
```

### 3. 内容呈现

```
由浅入深:
简介 → 概述 → 详细 → 技术细节

由总到分:
总览 → 分类 → 具体 → 案例
```
